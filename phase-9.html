Overview/Phase 09
Phase 09 Infrastructure as Code
Everything built by hand in AWS gets rebuilt as code. Not started
In one sentence: the AWS environment gets described as text (Terraform) so it can be destroyed and rebuilt identically, on demand.
Text becomes infrastructure
The only step that changes anything is the last one — and it never runs unread

How to build this — the code, not the clicks

Every task from here on is code, not console steps. This describes the Phase 1 network (HQ's VPC, subnets, IGW) as Terraform — the same pattern extends to the branch and cloud VPCs from Phases 5 and 8.

  • Show me the code
    variable "region" { default = "ap-northeast-3" }
    variable "vpc_cidr" { default = "10.10.0.0/16" }
    variable "public_subnet_cidr" { default = "10.10.1.0/24" }
    variable "private_subnet_cidr" { default = "10.10.20.0/24" }
  • Show me the code
    resource "aws_vpc" "hq" {
      cidr_block = var.vpc_cidr
      tags = { Name = "enterprise-lab-vpc" }
    }

    resource "aws_subnet" "public_wan" {
      vpc_id = aws_vpc.hq.id
      cidr_block = var.public_subnet_cidr
      tags = { Name = "public-wan" }
    }

    resource "aws_internet_gateway" "hq" {
      vpc_id = aws_vpc.hq.id
      tags = { Name = "enterprise-lab-igw" }
    }
  • Show me the code
    terraform {
      backend "s3" {
        bucket = "enterprise-lab-tfstate"
        key = "hq/terraform.tfstate"
        region = "ap-northeast-3"
        dynamodb_table = "terraform-locks"
      }
    }
  • Show me the code
    $ terraform init
    $ terraform plan -out=tfplan \\ read every line before continuing
    $ terraform apply tfplan
